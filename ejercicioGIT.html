Alumn@: CMRG

  1. ¿Que es un sistema de control de versiones?

    Un sistema de control de versiones es una herramienta que permite registrar, gestionar y organizar los cambios que se hacen en archivos a lo largo del tiempo, especialmente en proyectos de software. Te permite guardar el historial de cambios, volver a versiones anteriores y trabajar en equipo sin sobrescribir el trabajo de otros.

  2. ¿Cual es la diferencia entre git y github?

    Git → Es una herramienta que instalas en tu computadora para controlar versiones (guardar cambios, crear ramas, etc.). GitHub → Es una plataforma web donde alojas repositorios Git y colaboras con otras personas.

  3. ¿Cómo creo una rama en git?

    Se crea una rama con el siguiente comando: git branch [nombre-rama]

  4. ¿Cómo clono un repositorio?

    Se clona con el sigueinte comando: git clone [URL]

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Se bajan los últimos cambios de un repositorio con el siguiente comando: git pull

  6. ¿Cómo puedo gener un tag?

    - git tag v1.0.0 - git push origin v1.0.0

  7. Cómo hago una mezcla de ramas?

    - git checkout main - git merge otra-rama

  8. ¿Cómo puedo moverme a un commit especifico?

    git checkout [commit-id]
